Self-Contained Test Environment for AgenC
==========================================

Related beads: agenc-49fm, agenc-709q

Problem
-------

An agent working on the AgenC codebase cannot validate its own changes
end-to-end. Building the binary is straightforward, but running it creates
collisions with the user's real AgenC installation: tmux sessions fight over
the same names, launchd plists overwrite each other, and there is no structural
guarantee that the agent's built binary targets a test environment rather than
the real `~/.agenc`.

Goal
----

An agent can run `make build` then `make test-env` to get a fully isolated pocket
AgenC environment. It uses `_build/agenc-test` to run commands against this
environment. The test environment has its own database, server, tmux sessions,
and launchd registrations вЂ” all namespaced to avoid collisions with the real
installation or other test environments.

`_build/` and `_test-env/` have independent lifecycles. `make build` manages
`_build/`. `make test-env` and `make test-env-clean` manage `_test-env/`. Neither
touches the other's directory.

### Namespace resolver

New file: `internal/config/namespace.go`

`GetNamespaceSuffix(agencDirpath string) string` computes a deterministic suffix
from the resolved absolute path of `agencDirpath`:

- Default path (`~/.agenc`): returns `""` (empty string)
- Any other path: returns `-` + first 8 characters of SHA256 of the resolved
  absolute path (e.g., `-a1b2c3d4`)

All system-global resource names follow the pattern `agenc{-HASH}-{thing}`:

| Resource | Default | Pocket |
|----------|---------|--------|
| User tmux session | `agenc` | `agenc-a1b2c3d4` |
| Pool tmux session | `agenc-pool` | `agenc-a1b2c3d4-pool` |
| Cron plist prefix | `agenc-cron.` | `agenc-a1b2c3d4-cron.` |

### Tmux session names (dynamic)

`cmd/tmux_helpers.go` constant `tmuxSessionName = "agenc"` becomes a function
`GetTmuxSessionName(agencDirpath string) string`.

`internal/server/pool.go` constant `poolSessionName = "agenc-pool"` becomes a
function `GetPoolSessionName(agencDirpath string) string`.

Both call `GetNamespaceSuffix` internally.

### Launchd plist prefix (dynamic)

`internal/launchd/plist.go` constant `CronPlistPrefix = "agenc-cron."` becomes a
function `GetCronPlistPrefix(agencDirpath string) string`.

The cron syncer's orphan cleanup scan in `~/Library/LaunchAgents/` uses the
dynamic prefix to scope which plists it owns.

### AGENC_TEST_ENV feature gating

The `AGENC_TEST_ENV=1` environment variable marks a test environment. Features
that would conflict with the global installation **return errors** (not silent
skips) when this is set:

- Tmux keybinding injection (writes to `.tmux.conf`)

`config.IsTestEnv() bool` checks the environment variable. Code paths that touch
global state call this and return an error if true:

```
Error: tmux keybinding injection is disabled in test environments (AGENC_TEST_ENV=1)
```

Additional features can be gated as discovered.

### Wrapper script

`_build/agenc-test` is a bash script generated by `make build`:

```bash
#!/usr/bin/env bash
set -euo pipefail
script_dirpath="$(cd "$(dirname "${0}")" && pwd)"
export AGENC_DIRPATH="<absolute path to _test-env>"
export AGENC_TEST_ENV=1
exec "${script_dirpath}/agenc" "$@"
```

The agent uses `_build/agenc-test` for all test interactions. CLAUDE.md documents
this requirement.

### make test-env

1. Creates `_test-env/` with subdirectories: `config/`, `cache/`, `repos/`,
   `missions/`, `server/`, `logs/`, `stash/`
2. Runs `git init` in `_test-env/config/` (local-only, no remote)
3. Creates empty `_test-env/config/config.yml`
4. Creates `_test-env/config/claude-modifications/CLAUDE.md` (empty) and
   `settings.json` (`{}`)
5. Copies `~/.agenc/cache/oauth-token` to `_test-env/cache/oauth-token`
   (warns and skips if missing)
6. Prints summary

### make test-env-clean

Active cleanup вЂ” does not just remove the directory:

1. Reads `_test-env/server/server.pid`, sends SIGTERM if process exists
2. Computes namespace hash from absolute path of `_test-env/`
3. Kills tmux sessions: `tmux kill-session -t =agenc-HASH` and
   `tmux kill-session -t =agenc-HASH-pool` (ignores errors if absent)
4. Scans `~/Library/LaunchAgents/` for `agenc-HASH-cron.*` plists, runs
   `launchctl bootout` + deletes each
5. Removes `_test-env/`

Data flow
---------

### Namespace resolution

1. CLI command starts, calls `config.GetAgencDirpath()` to resolve the root
2. `config.GetNamespaceSuffix(agencDirpath)` computes hash (or empty string)
3. Callers that need system-global names use:
   - `config.GetTmuxSessionName(agencDirpath)` вЂ” used by cmd/tmux_helpers.go,
     cmd/tmux_attach.go
   - `config.GetPoolSessionName(agencDirpath)` вЂ” used by internal/server/pool.go
   - `launchd.GetCronPlistPrefix(agencDirpath)` вЂ” used by internal/launchd/plist.go,
     internal/server/cron_syncer.go

### Wrapper script invocation

1. Agent runs `_build/agenc-test mission ls`
2. Wrapper sets `AGENC_DIRPATH` and `AGENC_TEST_ENV=1`
3. Execs `_build/agenc` with all original args
4. `GetAgencDirpath()` picks up env var; everything targets `_test-env/`
5. Non-empty namespace suffix; tmux sessions and plists get hashed names

### AGENC_TEST_ENV gating

1. Code path reaches global-touching feature (e.g., tmux keybinding injection)
2. Calls `config.IsTestEnv()`
3. If true: returns error with clear message
4. Caller surfaces the error normally

Error handling
--------------

| Scenario | Behavior |
|----------|----------|
| `_build/agenc-test` run but `_test-env/` missing | Natural failure (server can't create socket, etc.) |
| `_build/agenc-test` run but `_build/agenc` missing | Wrapper fails: exec no such file |
| OAuth token missing during `make test-env` | Warn, skip copy |
| `make test-env-clean` but no `_test-env/` | No-op for each step, clean exit |
| `AGENC_TEST_ENV=1` + global feature | Error returned with message naming the gated feature |

Testing
-------

| Component | Approach |
|-----------|----------|
| `GetNamespaceSuffix()` | Unit: default returns `""`, custom returns deterministic 8-char hash |
| `GetTmuxSessionName()` / `GetPoolSessionName()` | Unit: verify `agenc{-HASH}-{pool}` pattern |
| `GetCronPlistPrefix()` | Unit: verify `agenc{-HASH}-cron.` pattern |
| `IsTestEnv()` | Unit: check env var |
| Makefile targets | Manual validation by agent using the test environment |
| Full round-trip | Agent: `make build`, `make test-env`, `_build/agenc-test mission new --headless`, verify in `_test-env/`, `make test-env-clean`, verify cleanup |
