Mission Detach CLI + Palette Command
=====================================

Context
-------

`agenc mission attach` links a mission's pool window into the current tmux session. There is no inverse: once attached, the only way to remove a mission window is to stop the mission or detach from tmux entirely. The user wants `mission detach` — unlink a mission window from the current session while leaving the wrapper running in the pool. This is the natural complement to attach.

The server-side handler (`handleDetachMission`) and API route (`POST /missions/{id}/detach`) already exist. What's missing is the client-side plumbing: the HTTP client method, the CLI command, and the palette entry.

Changes
-------

### 1. Client method — `internal/server/client.go`

Add `DetachMission(id, tmuxSession string) error` that POSTs to `/missions/{id}/detach` with a `DetachRequest` body. Mirrors `AttachMission` (line 298).

### 2. CLI command — `cmd/mission_detach.go` (new file)

`agenc mission detach [mission-id]`

Mirrors the structure of `cmd/mission_attach.go`:

- Requires tmux (calls `getCurrentTmuxSessionName()`)
- Lists running missions via `client.ListMissions(false, "")`
- Filters to missions whose pane is linked to the current session (new helper: `filterLinkedMissions`)
- Without args: shows fzf picker of linked missions
- With args: resolves mission ID, verifies it's linked, calls detach
- Calls `client.DetachMission(missionID, tmuxSession)`

**Filtering linked missions:** Build a set of pane IDs present in the current tmux session (via `tmux list-panes -s -t <session> -F "#{pane_id}"`), then keep only running missions whose `TmuxPane` is in that set. Add `filterLinkedMissions(missions, tmuxSession)` helper to `cmd/mission_helpers.go`.

### 3. Palette command — `internal/config/agenc_config.go`

Add `detachMission` entry to `BuiltinPaletteCommands`:

```go
"detachMission": {
    Title:       "🔌  Detach Mission",
    Description: "Unlink the focused mission from this tmux session (keeps running)",
    Command:     "agenc mission detach $AGENC_CALLING_MISSION_UUID",
},
```

Mission-scoped (contains `$AGENC_CALLING_MISSION_UUID`). No default keybinding — attach has one (`C-m`) but detach is less frequent.

Insert into `builtinPaletteCommandOrder` right after `switchMission` (attach and detach are natural pairs).

### 4. Auto-generated docs

`make build` regenerates CLI docs. A new `docs/cli/agenc_mission_detach.md` will be auto-created.
